# Python 리스트 내림차순 정렬 방법 정리

Python에서 배열(리스트)을 역순으로 정렬하는 대표적인 방법 3가지를 정리하고 비교한다.

---

### 1. `arr.sort(reverse=True)`
* **작동 방식**: **원본 배열 수정 (In-place Sort)**
* **설명**: 별도의 복사본을 만들지 않고 기존 `arr` 내부의 요소 순서를 직접 바꾼다. 메모리를 절약할 수 있어 효율적이다.
* **코드**:
    ```python
    arr = [5, 4, 6, 14]
    arr.sort(reverse=True)
    # 결과: [14, 6, 5, 4]
    ```

### 2. `arr = sorted(arr, reverse=True)`
* **작동 방식**: **새로운 배열 생성 (Return New List)**
* **설명**: 원본 리스트는 건드리지 않고, 내림차순으로 정렬된 새로운 리스트를 만들어 반환한다. 원본 데이터를 보존해야 할 때 사용한다.
* **코드**:
    ```python
    arr = [5, 4, 6, 14]
    arr = sorted(arr, reverse=True)
    # 결과: [14, 6, 5, 4]
    ```

### 3. `arr = sorted(arr)[::-1]`
* **작동 방식**: **정렬 후 슬라이싱 (Sorting then Slicing)**
* **설명**: `sorted(arr)`로 오름차순 정렬을 수행한 결과물에 `[::-1]`(슬라이싱)을 적용해 뒤집는다. (2)번보다 한 단계를 더 거치기 때문에 미세하게 성능 차이가 있을 수 있다.
* **코드**:
    ```python
    arr = [5, 4, 6, 14]
    arr = sorted(arr)[::-1]
    # 결과: [14, 6, 5, 4]
    ```

---

## 💡 요약 및 비교

| 방법 | 원본 데이터 | 성능 | 권장 상황 |
| :--- | :--- | :--- | :--- |
| **`arr.sort(reverse=True)`** | **변함 (Modified)** | 가장 빠름 | 메모리 절약이 필요할 때 |
| **`sorted(arr, reverse=True)`** | **유지 (Preserved)** | 빠름 | 원본 보존이 필요할 때 (**추천**) |
| **`sorted(arr)[::-1]`** | **유지 (Preserved)** | 보통 | 정렬 후 뒤집는 과정이 명시적으로 필요할 때 |

> **결론**: 단순히 내림차순 정렬이 목적이라면 코드 가독성과 의도가 명확한 **(1)번** 혹은 **(2)번** 방식을 사용하는 것이 가장 좋습니다.
